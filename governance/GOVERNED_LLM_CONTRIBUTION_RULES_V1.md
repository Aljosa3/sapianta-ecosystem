# GOVERNED_LLM_CONTRIBUTION_RULES_V1
Status: GOVERNED LLM CONTRIBUTION RULES
Layer: Governance Discovery

---

# 1. CONTRIBUTION PURPOSE

This artifact defines how LLM contributions may remain bounded,
governance-visible, replay-safe, and non-authoritative.

LLM contribution is participation, not authority.

---

# 2. PERMITTED CONTRIBUTIONS

An LLM may contribute:

- bounded proposal text;
- governance evidence summaries;
- replay-visible lineage explanations;
- non-authoritative semantic recommendations;
- review-oriented clarification drafts.

---

# 3. REQUIRED CONDITIONS

Every LLM contribution must remain:

- deterministic enough for replay visibility;
- attributable as LLM-originated participation;
- governance-bounded;
- non-executing;
- non-authoritative;
- fail-closed on ambiguity.

---

# 4. PROHIBITED CONTRIBUTIONS

LLM participation may not:

- authorize execution;
- mutate governance;
- bypass approval;
- orchestrate runtime behavior;
- self-promote recommendations;
- create hidden runtime memory;
- expand its own authority.

---

# 5. GOVERNANCE RELATIONSHIP

Governance may review, accept, reject, quarantine, or preserve LLM
contributions as evidence.

LLM output cannot approve itself.
