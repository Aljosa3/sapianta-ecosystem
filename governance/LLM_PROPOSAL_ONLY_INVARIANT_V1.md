# LLM Proposal Only Invariant V1

Status: permanent cognition authority boundary.

## Invariant

LLM / cognition participation is proposal-only.

An LLM may produce:

- a cognition proposal
- bounded semantic contribution
- requested capability target
- proposed arguments
- explanatory context

An LLM must never produce:

- execution authority
- authorization authority
- governance authority
- worker authority
- orchestration authority
- autonomous continuation authority

## Untrusted Proposal Input

Every LLM output must be treated as untrusted proposal input until AiGOL validates it.

The proposal must be normalized and checked before it can become a governed execution request. The proposal must fail closed if it contains:

- ambiguous intent
- missing capability target
- unsupported capability
- authority escalation language
- hidden continuation language
- mutation intent
- replay discontinuity
- malformed structure

## Prohibited Language And Semantics

Constitutional terminology must avoid implying that cognition executes.

Avoid:

- cognition executes
- LLM executes
- direct cognition-to-execution
- autonomous LLM execution
- LLM-driven execution

Prefer:

- cognition proposal
- untrusted proposal input
- proposal-to-execution-request bridge
- governed execution request
- deterministic worker execution
- AiGOL-governed authorization

## Certification

This invariant certifies that LLM participation remains:

- non-authoritative
- non-executing
- non-orchestrating
- non-self-authorizing
- replay-visible
- governance-mediated
