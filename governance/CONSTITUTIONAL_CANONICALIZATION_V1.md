# Constitutional Canonicalization V1

Status: post-consolidation canonicalization milestone.

This artifact establishes the first canonical constitutional language layer for the intelligence substrate. It is terminology canonicalization only and does not introduce capability expansion, orchestration runtime, new governance domains, additional lifecycle systems, or semantic feature growth.

## Purpose

The purpose is to ensure the constitutional intelligence substrate uses:

- one canonical replay language
- one canonical authority model
- one canonical boundedness language
- one canonical lifecycle vocabulary
- one canonical interpretation of repeated boundary guarantees

Canonicalization reduces semantic duplication. It does not add behavior.

## Canonicalization Rule

When a governance artifact repeats an existing concept, future work should:

1. Prefer the canonical term.
2. Reference the canonical definition.
3. Avoid parallel terminology unless a new failure mode requires it.
4. Mark redundant terms as aliases or deprecated candidates.

## Canonical Concept Families

### Replay

Canonical family:

- replay
- lineage
- continuity
- replay visibility
- replay integrity
- replay isolation

Replay is the single continuity spine of the substrate.

### Authority

Canonical family:

- authority
- non-authoritative participation
- governance authority
- runtime authority
- orchestration authority
- escalation semantics

Authority is explicit, separated, non-transferable by implication, and fail-closed on ambiguity.

### Boundedness

Canonical family:

- bounded
- fail-closed
- isolation
- containment
- termination
- non-persistence

Boundedness is the constitutional constraint that prevents hidden autonomy, hidden continuation, and capability drift.

### Lifecycle

Canonical family:

- lifecycle
- transitions
- states
- termination
- `FAILED`
- resurrection prohibition

Lifecycle semantics are replay-visible progression rules, not runtime execution.

## Review Findings

The substrate has repeated terminology across governance, replay, semantic continuity, cognition boundaries, LLM participation, runtime interaction, isolation, identity, and orchestration lifecycle artifacts.

The repetition is not blocking. It should now be compressed through canonical references.
