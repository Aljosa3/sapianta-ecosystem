## Status

Review-only recommendations.

## Primary Recommendation

Prioritize native development readiness for one corrective milestone before opening the first Trading Worker through AiGOL conversation mode.

Recommended path:

```text
Prioritize Native Development Readiness
```

## Highest-Value Corrective Milestone

Create:

```text
AIGOL_NATIVE_DEVELOPMENT_TASK_INTAKE_AND_SESSION_RESUME_V1
```

Minimum scope:

- durable append-only conversation turn allocation;
- session resume visibility;
- native development intent classification;
- target milestone extraction;
- domain and worker foundation resolution;
- deterministic context assembly references;
- provider necessity decision;
- fail-closed ambiguity handling;
- read-only task-intake artifact;
- suggested next commands for inspection and implementation handoff.

## Why This Milestone Is Smallest Useful Work

Fixing only the append-only collision would allow more turns, but would not solve ambiguous development prompts.

Fixing only provider availability would allow more proposals, but would not make development routing deterministic.

The smallest useful correction is a combined intake and session-resume foundation that allows AiGOL to say deterministically:

- what development task was requested;
- which domain and worker are targeted;
- which artifacts are relevant;
- whether provider assistance is needed;
- whether the task can proceed or must fail closed.

## Guardrails

The corrective milestone must not:

- create execution requests;
- dispatch workers;
- invoke workers;
- mutate governance;
- mutate replay outside append-only evidence;
- bypass human authorization;
- treat provider text as authority.

## Trading Recommendation

Pause Trading Worker implementation through conversation mode until the corrective milestone exists.

Trading architecture work may continue through explicit, human-directed Codex development if it remains:

- read-only where review-only;
- evidence-producing where runtime work is requested;
- no broker integration;
- no exchange integration;
- no order placement;
- no live trading;
- no financial performance claims.

## Final Recommendation

Prioritize native development readiness now, then resume:

```text
TRADING_MARKET_EVIDENCE_NORMALIZATION_WORKER_FOUNDATION_V1
```

This avoids turning provider ambiguity and session replay collisions into recurring domain-development friction.
