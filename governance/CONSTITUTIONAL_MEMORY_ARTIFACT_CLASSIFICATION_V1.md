# Constitutional Memory Artifact Classification V1

Status: artifact category classification for Constitutional Memory.

## CORE

Core artifacts define constitutional meaning.

Includes:

- constitutional architecture spec
- constitutional invariants
- canonical layer model
- governance enforcement hierarchy
- governance lineage model
- canonical replay language
- canonical authority model
- constitutional baseline freezes
- first useful AiGOL baseline and guarantees
- operational epoch freeze artifacts

Justification: these artifacts define what must be preserved.

## SUPPORTING

Supporting artifacts provide evidence that the core was applied.

Includes:

- acceptance artifacts
- certification artifacts
- manifest artifacts
- governance reviews
- boundary guarantees
- position reviews
- readiness reviews
- pressure validations
- stability reports

Justification: these artifacts certify, explain, or reconstruct core meaning, but do not override it.

## DERIVED

Derived artifacts are views or outputs from replay, runtime, or reconstruction.

Includes:

- runtime replay reconstruction outputs
- `.runtime/` operational evidence
- lineage summaries
- replay summaries
- governed result summaries
- runtime hashes and evidence chains

Justification: these artifacts help reconstruct events and lineage, but they remain subordinate to core constitutional artifacts and supporting certifications.

## OUT_OF_SCOPE

Out-of-scope artifacts are not Constitutional Memory.

Includes:

- vector stores
- semantic search indexes
- autonomous memory stores
- hidden runtime state
- provider conversation memory
- worker memory
- orchestration memory
- unstaged local scratch output not tied to governance or replay

Justification: these either do not exist in the current constitutional memory model or would introduce unacceptable authority and duplication risk.
