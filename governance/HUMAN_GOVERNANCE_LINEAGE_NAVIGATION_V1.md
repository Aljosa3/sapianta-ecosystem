# 3. HOW TO FOLLOW LINEAGE

Read each milestone as preserving or clarifying the previous boundary.

Do not read a later artifact as silently replacing an earlier artifact.

If a later artifact supersedes a prior artifact, it must say so
explicitly.

If lineage appears ambiguous, preserve the prior constitutional boundary
and fail closed at the interpretation level.

---

# 4. GOVERNANCE EVOLUTION INTERPRETATION

Governance evolution should be read as bounded clarification of
authority, replay evidence, promotion requirements, rollback
requirements, and operational evidence continuity.

Governance evolution does not imply autonomous governance mutation.

---

# 5. REPLAY-SAFE PROGRESSION

Replay-safe progression means evidence remains deterministic,
append-only, traceable, and operator-readable.

Replay evidence can support interpretation, but replay verification does
not authorize mutation or promotion by itself.

---

# 6. NON-ACTIVATION

This navigation guide does not implement lineage traversal, runtime
memory, governance reasoning, graph queries, orchestration, or mutation
authority.
