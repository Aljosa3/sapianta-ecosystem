# AIGOL_DEVELOPMENT_CONTEXT_ASSEMBLY_BOUNDARY_GUARANTEES_V1

## Status

Review-only boundary guarantees.

## Core Boundary

Development context assembly is not development execution.

It only determines:

```text
Which certified context should accompany a native development task?
```

## Authority Guarantees

The context assembly model preserves:

```text
LLM proposes
AiGOL governs
Human authorizes
Worker executes
Replay records
```

## Forbidden Effects

Context assembly must not:

- create governance files;
- modify governance;
- create domains;
- create workers;
- create implementation plans;
- create execution requests;
- dispatch;
- invoke;
- execute;
- place orders;
- integrate brokers;
- integrate exchanges;
- make financial claims;
- mutate replay outside append-only evidence.

## Provider Boundary

Providers may only receive context after deterministic assembly.

Providers must not:

- select authoritative context;
- resolve ambiguity;
- fill missing governance;
- authorize implementation;
- create execution requests;
- mutate replay;
- invoke workers.

Provider text is proposal-only.

## Domain Boundary

Domain artifacts provide domain-specific context.

They do not redefine:

- AiGOL Core governance;
- authority boundaries;
- replay semantics;
- provider role;
- worker role;
- human authorization requirements.

## Trading Boundary

For Trading Domain, context assembly must preserve:

- no broker integration;
- no exchange integration;
- no order placement;
- no live trading;
- no autonomous trading;
- no profitability claims;
- no financial advice claims.

These boundaries apply even when the milestone is a worker foundation.

## Future Domain Boundary

For future Marketing, Healthcare, and Public Services domains, context assembly must use domain-specific constraints without weakening core governance.

Future domain context must be explicit and certified before being treated as required context.

## Fail-Closed Guarantees

Context assembly must fail closed on:

- invalid intake;
- missing mandatory context;
- ambiguous domain;
- ambiguous worker family;
- conflicting certifications;
- missing artifact hash;
- invalid replay reference;
- authority-bearing request;
- provider attempt to determine context.

## Human Boundary

Human authority remains required before any implementation, approval, bridge authorization, worker creation, dispatch, or execution occurs.

Context assembly may suggest a next safe handoff, but it cannot authorize that handoff.
