# AiGOL Governance Authority Invariant V1

Status: permanent governance authority invariant.

## Invariant

AiGOL is the governance authority, validator, authorizer, controller, replay recorder, and constitutional constraint enforcer.

Neither LLM nor worker owns constitutional authority.

## AiGOL Must Govern

AiGOL must govern:

- proposal normalization
- constitutional validation
- Layer 0-4 governance checks
- capability boundary validation
- authorization or rejection
- replay lineage recording
- execution evidence recording
- governed return creation

## Layer 0-4 Governance Checks

Governed execution request creation must preserve:

- Layer 0 constitutional immutability
- Layer 1 canonical artifact stability
- Layer 2 decision spine restrictions
- Layer 3 governance system constraints
- Layer 4 bounded research and evolution constraints

No proposal or worker action may bypass these checks.

## Authorization Boundary

Authorization must be explicit, replay-visible, deterministic, and bounded.

Authorization must not be:

- inferred from LLM wording
- inferred by the worker
- delegated implicitly
- inherited across sessions
- created after execution
- hidden from replay

## Replay Authority

Replay remains mandatory and authoritative for:

- proposal input
- normalized request
- validation
- authorization or rejection
- worker execution evidence
- fail-closed artifacts
- governed return

## Certification

This invariant certifies:

- AiGOL remains governance authority
- LLM remains proposal-only
- worker remains execution-only
- replay remains mandatory
- authority separation remains permanent
