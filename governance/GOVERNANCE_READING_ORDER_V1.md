# GOVERNANCE_READING_ORDER_V1
Status: CANONICAL GOVERNANCE READING ORDER
Layer: Governance Discovery
Principle: Stabilization Before Expansion

---

# 1. PURPOSE

This artifact defines the recommended human reading order for AiGOL
governance artifacts.

It is documentation only. It is not a semantic navigation system,
adaptive index, runtime governance memory, or governance engine.

---

# 2. RECOMMENDED READING ORDER

1. `governance/README.md`

   Start here for governance purpose, topology, boundaries, and human
   orientation.

2. `governance/GOVERNANCE_ENTRYPOINT_ORIENTATION_V1.md`

   Read this for concise entrypoint orientation and role separation.

3. `governance/EVOLUTION_GOVERNANCE_INDEX_V1.md`

   Read this for the deterministic top-level governance map.

4. Evolution governance.

   Read `governance/evolution/` for EPP, mutation boundaries, replay
   requirements, promotion requirements, rollback guarantees, bounded
   refinement, anti-overengineering, and expansion criteria.

5. Constitutional governance.

   Read `governance/constitutional/gep/`,
   `governance/constitutional/replay/`, and
   `governance/constitutional/promotion/` for governance evolution,
   replay evidence, lineage contracts, promotion gates, and rollback
   requirements.

6. Operational governance.

   Read `governance/operational/` for replay stabilization evidence,
   fail-closed expectations, replay chain failure-mode notes,
   operational evidence indexing, and semantic hardening records.

7. Governance topology and lineage.

   Read `governance/GOVERNANCE_TOPOLOGY_MAP_V1.md`,
   `governance/GOVERNANCE_DISCOVERY_STRUCTURE_V1.md`, and
   `governance/GOVERNANCE_ARTIFACT_LINEAGE_MAP_V1.md`.

8. Human interpretation guidance.

   Read `governance/GOVERNANCE_EVIDENCE_READING_GUIDE_V1.md`,
   `governance/GOVERNANCE_INTERPRETATION_RULES_V1.md`, and
   `governance/HUMAN_GOVERNANCE_LINEAGE_NAVIGATION_V1.md`.

---

# 3. STABILIZATION-BEFORE-EXPANSION RULE

When a reader encounters operational ambiguity, the interpretation
default is bounded semantic refinement, deterministic hardening, and
fail-closed clarification before architecture expansion.

---

# 4. NON-ACTIVATION

This reading order does not implement runtime behavior, governance
engines, orchestration, adaptive interpretation, runtime governance
memory, or governance authority.
