## Dispatch Guarantees

Dispatch semantics are bounded:

- dispatch preview is not authorization
- dispatch request is not authorization

- runtime dispatch is bounded single-dispatch recording only
- runtime dispatch is not provider execution

## Runtime Guarantees

Runtime dispatch recording is explicit, deterministic, lineage-linked, and
fail-closed. It does not activate providers, complete execution, call external
systems, retry automatically, recurse, or create orchestration.

## Provider Execution Guarantees

Provider execution gate semantics are eligibility-only. The gate does not
execute providers, call APIs, run shell commands, activate external systems,
produce execution results, retry, recurse, or orchestrate.

## Human Approval Guarantees

Human approval remains explicit and bounded. Approval supports downstream
worker preparation eligibility only and does not become dispatch, provider
activation, execution, or autonomous continuation.

## Fail-Closed Guarantees

Missing, malformed, invalid, or mismatched evidence must fail closed or remain
explicitly incomplete according to the relevant layer. MOC V1 must not silently
repair gaps, infer hidden evidence, or proceed through unknown state.

## Deterministic Operational Guarantees

MOC V1 operational artifacts use deterministic artifact construction,
deterministic hashes, explicit timestamps, visible violations, visible
warnings, visible unknowns, and replay-visible references.
