# MOC_V1_ADVISORY_PROTOCOL

Status: advisory protocol specification.

## Purpose

The MOC V1 advisory protocol defines how bounded operational cognition may produce governance-visible proposals without gaining authority.

## Advisory Inputs

Allowed inputs:

- explicit human intent
- explicit governance anchors
- replay-visible cognition artifacts
- semantic context state
- authority propagation evidence
- governed return evidence

Hidden context ingestion is forbidden.

## Advisory Processing

MOC V1 may perform:

- intent normalization
- governance anchor retrieval
- semantic contract construction
- scope classification
- risk classification
- mutation classification
- advisory next-step proposal formation
- governed return interpretation

Processing must remain deterministic where specified, bounded, and replay-visible.

## Advisory Outputs

Allowed outputs:

- semantic contracts
- advisory proposals
- bounded worker task proposals
- governance artifact proposals
- return interpretation summaries

Outputs are non-authoritative until separately approved and authorized through governed boundaries.

## Prohibited Outputs

MOC V1 must not output:

- executable plans
- self-dispatch instructions
- hidden continuation tokens
- provider activation requests
- governance mutations
- authority grants
- replay repairs
- semantic truth certifications

## Required Contract Flags

Every MOC V1 semantic contract must include:

- `advisory_only: true`
- `replay_safe: true`

## Human Authority

Human approval remains a distinct governance boundary. Advisory output may be accepted, rejected, or revised by human authority, but advisory output never becomes authority by itself.
