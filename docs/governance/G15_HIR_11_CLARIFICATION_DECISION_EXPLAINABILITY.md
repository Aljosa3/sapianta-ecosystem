## Architectural Review

Clarification decision explainability is owned by Platform Core.

The Human Interface boundary remains unchanged:

```text
Human Interface captures answer
-> Platform Core verifies active semantic slot
-> Platform Core records decision explainability
-> Human Interface renders Platform Core explanation
```

AiCLI does not infer why an answer was rejected. It only renders the `human_conversation_experience` produced by Platform Core.

## Clarification Explainability Review

Platform Core now records semantic-only explanation evidence when evaluating an active clarification answer.

The explanation identifies:

- active semantic slot;
- active question text;
- expected semantic outcome;
- accepted semantic requirements;
- unresolved semantic requirements;
- deterministic continuation status;
- deterministic continuation reason;
- concise user-facing explanation.

The explanation avoids implementation internals and exposes only semantic reasoning.

When the answer is insufficient, Platform Core no longer gives only a generic message. It reports what was accepted and what is still required before governed continuation can proceed.
