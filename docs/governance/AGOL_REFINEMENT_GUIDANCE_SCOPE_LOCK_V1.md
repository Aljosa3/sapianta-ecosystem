# AGOL Refinement Guidance Scope Lock V1

Status: finalized scope lock declaration.

Workflow:
`AGOL_REFINEMENT_GUIDANCE_WORKFLOW_V1`

## Locked Scope

The workflow is strictly limited to:

- refinement guidance;
- refinement recommendation;
- prompt augmentation assistance;
- bounded escalation suggestion;
- refinement mode interpretation.

## Locked Non-Execution Rule

The workflow does not execute commands, mutate files, start runtime processes,
deploy software, or create orchestration authority.

The workflow may generate guidance only.

## Locked Proposal-Only Rule

The workflow may:

- recommend a refinement mode;
- explain why a refinement mode is appropriate;
- propose a bounded refinement direction;
- generate prompt augmentation text;
- identify when higher-impact bounded refinement should be considered.

The workflow may not:

- mutate Product 1 automatically;
- redesign autonomously;
- execute runtime actions;
- generate orchestration authority;
- bypass governance continuity;
- override human approval.

## Controlled Changes

The following require governed review and renewed approval:

- adding automatic Product 1 mutation;
- adding runtime execution;
- adding browser or server orchestration;
- adding deployment behavior;
- adding autonomous redesign loops;
- changing the user final authority requirement;
- changing the replay identity fields;
- expanding the workflow beyond proposal-only guidance.

## Non-Inheritance Rule

Certification of this workflow does not authorize:

- UI generation;
- runtime mutation;
- autonomous product redesign;
- deployment automation;
- unrestricted prompt rewriting;
- self-directed execution.

All future use remains bounded by explicit user-directed task scope.
