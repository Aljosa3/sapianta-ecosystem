# AIGOL_CONVERSATIONAL_DISPATCH_AUTHORITY_UNIFICATION_V1

## Status

Implementation milestone.

The conversational CLI now uses the certified conversational routing decision and workflow selection artifacts as the ordinary prompt dispatch authority.

Final classification:

```text
DISPLAYED_WORKFLOW_EQUALS_EXECUTED_WORKFLOW = true
```

## Implemented Architecture

Authoritative ordinary prompt path:

```text
Human Prompt
-> Stateful Pre-Routing Gate
-> route_conversational_cli_intent(...)
-> CONVERSATIONAL_ROUTING_DECISION_ARTIFACT_V1
-> CONVERSATIONAL_WORKFLOW_SELECTION_ARTIFACT_V1
-> dispatch by workflow_selection_artifact["workflow_id"]
-> routing visibility projected from authoritative selection
-> workflow runtime
-> result
```

`CONVERSATIONAL_ROUTING_VISIBILITY_ARTIFACT_V1` remains visibility-only. It is not dispatch authority.

## Runtime Changes

### Interactive CLI

Updated:

```text
aigol/cli/aigol_cli.py
```

Implemented:

- stateful pre-routing gates remain before ordinary routing;
- ordinary prompts call `route_conversational_cli_intent(...)` once;
- ordinary workflow dispatch uses `workflow_selection_artifact["workflow_id"]`;
- routing visibility for ordinary prompts is generated from the authoritative routing capture;
- OCS prompts selected by authoritative routing execute the OCS end-to-end branch;
- operator-support prompts selected by authoritative routing execute the operator-support branch;
- default provider-assisted conversation is explicit routing authority, not an implicit unclassified fallthrough.

Stateful gates preserved:

- implementation approval resume;
- human approval reject/request-modification;
- recommendation approval;
- recommendation follow-up.

### Certified Router

Updated:

```text
aigol/runtime/conversational_cli_runtime.py
```

Added workflow registry coverage for:

- `NATIVE_DEVELOPMENT_CONTEXT_INTEGRATION`;
- `DEFAULT_PROVIDER_ASSISTED_CONVERSATION`.

Adjusted routing priority so broad OCS cognition can become authoritative before operator decision-support classification when the prompt matches OCS cognition markers.

The router still fails closed for prompts requesting unrestricted autonomous-agent behavior.

### Routing Visibility

For ordinary prompts, routing visibility now projects:

- `workflow_id`;
- `routing_confidence`;
- `matched_signals`;
- `routing_reason`;

from the authoritative routing decision and workflow selection artifacts.

Competing signals are no longer generated by a parallel display-only classifier for ordinary prompts.

## Dispatch Guarantees

For ordinary prompts:

```text
route_conversational_cli_intent(...) call count = 1
dispatch source = workflow_selection_artifact["workflow_id"]
displayed workflow = executed workflow
```

For stateful prompts:

```text
stateful gate remains authoritative before ordinary routing
```

This preserves session-bound approval and recommendation semantics.
